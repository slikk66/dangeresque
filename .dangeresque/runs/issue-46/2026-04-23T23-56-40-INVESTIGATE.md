## Proposed Implementation

### New file: `src/picker.ts`

```ts
import { createInterface } from "node:readline";
import type { Readable, Writable } from "node:stream";
import type { WorktreeInfo } from "./worktree.js";

export interface PickerOptions {
  input?: Readable;
  output?: Writable;
  isTTY?: boolean;
  label?: string;
}

export async function pickWorktree(
  worktrees: WorktreeInfo[],
  opts: PickerOptions = {},
): Promise<WorktreeInfo | undefined> {
  const input  = opts.input  ?? process.stdin;
  const output = opts.output ?? process.stderr;
  const isTTY  = opts.isTTY  ?? (input as NodeJS.ReadStream).isTTY === true;

  if (!isTTY || worktrees.length === 0) return undefined;

  output.write(`${opts.label ?? "Select a worktree"}:\n`);
  for (let i = 0; i < worktrees.length; i++) {
    const wt = worktrees[i];
    const short = wt.branch.replace(/^worktree-dangeresque-/, "");
    const state = wt.running ? "RUNNING" : "IDLE";
    output.write(`  ${String(i + 1).padStart(2)}. ${short.padEnd(20)} ${state}\n`);
  }

  const rl = createInterface({ input, output, terminal: false });
  try {
    const answer: string = await new Promise((resolve) => {
      rl.question(`Selection [1-${worktrees.length}]: `, resolve);
    });
    const n = parseInt(answer.trim(), 10);
    if (!Number.isInteger(n) || n < 1 || n > worktrees.length) return undefined;
    return worktrees[n - 1];
  } finally {
    rl.close();
  }
}
```

Key design choices, each deliberate:

- **`output` defaults to `process.stderr`** — picker prompt never pollutes stdout if the next command pipes.
- **`terminal: false`** — lets the terminal handle character echo in canonical mode; avoids raw-mode complexity and makes PassThrough-based tests trivial. Ctrl-C falls through to the default SIGINT handler → exit 130, which is acceptable cancel UX.
- **Single-shot input** (no re-prompt on invalid) — keeps logic linear, keeps tests synchronous, and matches "small stdlib-based picker" scope. Users who mistype can rerun.
- **Returns `undefined` in four cases** (non-TTY, empty list, invalid input, cancel via Ctrl-D/empty) — caller falls through to the pre-existing error message. Uniform failure mode.
- **`isTTY` is injectable** — tests pass `isTTY: true` with PassThrough streams.

### Helper in `src/worktree.ts`

```ts
export type WorktreeFilter = "all" | "running" | "finished";

export function filterWorktrees(
  worktrees: WorktreeInfo[],
  filter: WorktreeFilter,
): WorktreeInfo[] {
  if (filter === "all")     return worktrees;
  if (filter === "running") return worktrees.filter((w) =>  w.running);
  return worktrees.filter((w) => !w.running);
}
```

Worth adding (vs inlining) because (a) the filter is called from four sites, (b) the named type documents intent, (c) it's unit-testable in isolation.

### Wiring (`src/cli.ts`)

Per-command filter:

| Command | Filter     | Rationale                                                |
|---------|------------|----------------------------------------------------------|
| logs    | `all`      | IDLE runs still have transcripts to inspect              |
| results | `all`      | IDLE runs have the final artifact; running ones have WIP |
| merge   | `finished` | Can't merge a branch whose worker is still writing to it |
| discard | `finished` | Safer default; arg-form still lets power users discard running ones |

Example (`cmdLogs`):

```ts
if (!positional) {
  const picked = await pickWorktree(filterWorktrees(worktrees, "all"), {
    label: "Select a worktree to view logs for",
  });
  if (picked) {
    // proceed with picked.branch
  } else {
    console.error(formatMissingTargetError("logs", "<branch>", worktrees));
    process.exit(1);
  }
}
```

## Observations

- **`formatMissingTargetError` already exists** and is used by `cmdLogs` + `cmdResults` (`src/cli.ts:643`). `cmdMerge`/`cmdDiscard` print a simpler "Run 'dangeresque status'" message — adopting `formatMissingTargetError` for them would be a nice consistency win but is out of scope for this issue.
- **`readline` is already imported in `src/logs.ts:3`** (`createInterface`), so the codebase has precedent for the exact API we need.
- **`isTTY` semantics**: `process.stdin.isTTY === true` on real terminals, `undefined` when piped/redirected. The strict `=== true` check is safe (avoids accidental truthy on a non-bool value).
- **Async propagation**: Main dispatch (`src/cli.ts:143-194`) already awaits `cmdRun` and `cmdLogs`. Adding `await` to the other three cases is a one-line change each.

## Risks / Uncertainty

1. **Signature change for `cmdMerge`/`cmdDiscard`**: from `(branch: string | undefined)` to `(args: string[])`. Low risk — these are internal functions not exported from `src/index.ts`.
2. **Filter choice for `logs` is ambiguous** in the issue ("all worktrees (or filter to running)"). I recommend **"all"** because (a) it matches `dangeresque status`, (b) IDLE worktrees still have readable transcripts, (c) users often want to read logs of a just-finished run.
3. **Filter choice for `discard`**: picking only finished is safer but prevents picker-discarding a runaway worker. Power users can still use arg-form (`dangeresque discard <branch>`) to bypass the filter. Fine tradeoff.
4. **Terminal echo on non-raw mode**: On some exotic terminals / CI with allocated pseudo-TTY but canonical mode disabled, the user may not see their own input. In practice (bash/zsh/iTerm/Terminal/tmux), canonical mode is default and echo works. Not a concern for the target audience.
5. **Ctrl-C handling**: With `terminal: false`, Ctrl-C goes to process default handler and kills the CLI with exit 130. No cleanup needed because we haven't mutated anything yet. Good enough; if fancier cancel messaging is desired, add `rl.on("SIGINT", ...)` later.
