## Filter Behavior

Filters compose (applied as AND after the schema gate):

- `--issue <N>` — numeric; matches `issue_number === N`
- `--engine <name>` — must be `claude` or `codex`; matches `engine === name`
- `--mode <mode>` — case-upper-folded; matches `mode === MODE`
- Unknown flag (e.g. `--foo`) → error + exit 1
- `--issue abc` → error + exit 1
- `--engine bogus` → error + exit 1

Zero-match case: prints full output with zero counters for all known sections (stable shape for scripts) + a `Note: no artifacts match filters (<flags>)` line, per staged Q3/Q5 guidance.

Empty `.dangeresque/runs/` (or missing dir): prints `No run artifacts found at <path>`, exits 0, per issue requirement.

## Design Notes

- **Median/p95 conventions**: median uses the standard n-odd → middle / n-even → rounded-avg-of-two-middles definition. p95 uses nearest-rank (`Math.ceil(p*n) - 1`, clamped). On small n (24 items) both give stable, defensible numbers.
- **Path display**: `formatStats` accepts whatever `runsDir` string the caller passes; `cmdStats` supplies `path.relative(projectRoot, runsDir)` so the `Path:` line stays under 80 cols even on deeply-nested filesystems. The lib does not know about `projectRoot`; only the CLI does.
- **Engine display**: always shows `claude` and `codex` with zero-fill, even when absent (per staged Q3). Extra engines (none today) would be appended alphabetically. This keeps output shape stable for scripts.
- **Mode ordering**: by count desc, alpha tiebreak. Model ordering: same. Results/Verdicts/Engines have fixed canonical order matching the issue's enum order.
- **Duration units**: raw ms with locale thousand-separators (`199,758`), per staged Q2. Pretty-printing (seconds/minutes) is a future polish.
