## Risks / Uncertainty

1. **Schema version drift** — see Challenge above. Primary risk is that a naive reading of the issue ("only v1") breaks on real data. Mitigation: accept both v1 and v2.
2. **Codex artifacts absent from corpus** — the "By engine" row will only show `claude` until codex runs ship. Not a blocker; the formatter must still emit the `By engine` section unconditionally (show `0` for missing engines? or only engines present? — issue says "count per engine (`claude`, `codex`)", which I read as "show both, zero-fill absent". Recommend zero-fill for the two known engines, skip unknowns.)
3. **`review_model` diversity** — all current IMPLEMENT artifacts have `review_model == model == "claude-opus-4-7[1m]"`. Issue #16 (reviewer model separation) is actually CLOSED (not "pending" as issue #19 claims), but no one has actually run with `--review-model` set to something different yet. "By model" will be uniform until someone does.
4. **Model field inclusion** — issue's "By model" wording is ambiguous between worker model and reviewer model. Recommend: aggregate BOTH into one bucket (i.e., count the worker `model` field only, since that's the primary run model; `review_model` is a follow-on). If someone wants reviewer-model breakout separately later, add a `--breakdown reviewer` flag in a future issue.
5. **Duration units** — durations in artifact are ms. Median/p95 output in ms is correct but verbose; consider printing ms with thousand-separators OR seconds with 1 decimal. The issue doesn't specify; ms with commas is simplest and unambiguous.
6. **Percentile definition for p95 with small N** — 23 artifacts is small. Use the straightforward "take element at index `Math.ceil(0.95 * n) - 1` of sorted array" (R-7-ish). Document in comment if needed.
7. **No-issue-number runs** — `RunArtifact.issue_number` is `number | null`. Fixture-driven runs have `null`. When `--issue N` is passed, null-issue artifacts should be excluded (obvious). When no filter, they should be included. Expect zero in current data.
8. **File discovery scope** — issue says `.dangeresque/runs/**/*.json` (glob). Actual tree is exactly 2 levels (`runs/issue-N/file.json`). Recommend walking just these two levels explicitly rather than a generic recursive walk, to avoid accidentally picking up anything stray (e.g. a user-dropped `.json` note).
