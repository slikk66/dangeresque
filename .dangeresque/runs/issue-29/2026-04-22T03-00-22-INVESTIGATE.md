## Recommended Implementation

Add a private helper in `src/stats.ts`, for example `formatSummary(summary: StatsSummary): string[]`, and call it from `formatStats` before `Results:`.

Recommended shape:

```text
Summary
-------
Overall success:     79.3% (23/29)
Hard failures:       1
Review coverage:     48.3% (14/29 verdicts)
Reviewer verdicts:   13 accept, 1 reject, 0 needs_human_review
Top failure category: scope_violation (5)
```

Exact wording is implementation discretion, but every number above is derivable from `StatsSummary`:

- Overall success: `summary.byResult.success / summary.total`.
- Hard failures: `summary.byResult.failure`.
- Review coverage: `accept + reject + needs_human_review` over `summary.total`. I recommend excluding `skipped` and `unknown` here because `unknown` can mean worker failure, missing artifact, or unparseable markdown; it is not always proof that a useful review verdict exists.
- Reviewer verdicts: the three concrete reviewer decisions, `accept`, `reject`, and `needs_human_review`.
- Top failure category: first entry from `Object.entries(summary.failureCategories)` sorted by count desc, then name asc, matching the existing Failure categories table sort at `src/stats.ts:266-268`; render `(none)` when empty.

Use the existing `formatPct` helper. For zero artifacts, avoid division by zero and render factual zero values, e.g. `0.0% (0/0)` and `Top failure category: (none)`.

Placement recommendation:

- Keep the existing title and metadata lines (`Run Evaluation Stats`, path, total, schema versions, filters, skipped, zero-match note).
- Insert a blank line and `Summary` / `-------` before `Results:`.
- This satisfies "Summary section is first in output, before Results" while preserving useful run-scope metadata above all analytical sections.
