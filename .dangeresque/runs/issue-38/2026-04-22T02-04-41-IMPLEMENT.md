## Changes

### 1. `src/runner.ts`

Added `commitWorkerChanges(worktreePath, issueNumber, mode)`:

```typescript
export function commitWorkerChanges(
  worktreePath: string,
  issueNumber: number,
  mode: string
): void {
  try {
    execSync(`git add -A -- ':(exclude).dangeresque/runs'`, {
      cwd: worktreePath, encoding: "utf-8", stdio: "pipe",
    });
    const staged = execSync("git diff --cached --name-only", {
      cwd: worktreePath, encoding: "utf-8", stdio: "pipe",
    }).trim();
    if (!staged) return;
    const message = `codex ${mode} worker: issue #${issueNumber}`;
    execSync(`git commit -m "${message}"`, {
      cwd: worktreePath, encoding: "utf-8", stdio: "pipe",
    });
  } catch {
    // Nothing to commit or transient git error — commitArchiveFile runs after
    // and reports its own outcome.
  }
}
```

Key design choices:
- **Pathspec exclude `:(exclude).dangeresque/runs`** scopes `git add -A` to everything EXCEPT the run-artifact directory. Preserves the existing two-commit semantic: worker-code commit → `commitArchiveFile` artifact commit. If the exclude pathspec ever silently fails, the artifact collapses into the worker-code commit with no correctness loss — `commitArchiveFile`'s try/catch swallows the "nothing to commit" exit. Low-blast-radius failure mode.
- **`git diff --cached --name-only` pre-commit check** avoids git's "nothing to commit, working tree clean" non-zero exit from the commit step. Keeps logs clean when a codex worker exits without touching any file (e.g. INVESTIGATE-mode codex run that only wrote the artifact).
- **Commit message `codex ${mode} worker: issue #${issueNumber}`** — terse, honest, searchable. Worker's actual deliverable is the DIFF, which the reviewer examines directly.

Wired into the codex branch `child.on("close")` block, gated on exit code 0 and BEFORE `commitArchiveFile`:

```typescript
child.on("close", (code: number | null) => {
  logStream.end();
  removePidFile(worktreePath);
  if ((code ?? 0) === 0) {
    commitWorkerChanges(
      worktreePath,
      opts.issueData.number,
      opts.mode ?? "INVESTIGATE"
    );
    commitArchiveFile(worktreePath, archivePath);
  }
  resolve({ worktreeName, branch, exitCode: code ?? 0, archivePath });
});
```

Only the codex branch was modified. The claude branch (`child.on("close")` near line 573) is unchanged — claude workers commit themselves with richer messages from inside the worktree, and re-running `commitWorkerChanges` against a fully-committed claude worktree would be a silent no-op anyway.

## Risks / Uncertainty

1. **Commit message authenticity.** The commit message is dangeresque-authored, not worker-authored. Reviewers still verify against the DIFF (which IS worker-authored), so this is cosmetic. If future iterations need richer messages, the worker's run-result SUMMARY block could be parsed and used — not done here to keep the change minimal.

2. **`git add -A` semantic breadth.** On the off chance a codex worker creates a stray file outside `.gitignore` that it didn't intend, `-A` pulls it in. Mitigation: reviewer's scope check catches this. If this becomes a recurring issue, a future iteration could read codex's JSONL session log for `file_change` items and stage precisely those.

3. **Silent failure of the exclude pathspec.** `:(exclude)` is git pathspec magic; if an extremely old git version doesn't support it, the exclude is ignored and the artifact gets bundled into the worker-code commit. Result: one commit instead of two, but no information loss. Unlikely to occur in practice — pathspec magic has been stable in git for >8 years.

4. **Exit-code-0 gating.** Only commits when codex exits cleanly. If codex exits non-zero but the worker DID make legitimate progress, those changes won't be salvaged. This mirrors the existing `commitArchiveFile` behavior — if the worker crashed mid-edit, salvaging partial state is usually the wrong default. The worktree is left intact for human inspection.

5. **Mode ambiguity in my own prompt.** Issue body says INVESTIGATE; initial prompt says IMPLEMENT. Chose IMPLEMENT on the strength of the pass-2 INVESTIGATE's explicit recommendation. If the human wanted a third INVESTIGATE instead, `dangeresque discard worktree-dangeresque-implement-38` reverts cleanly.

6. **Exclude pattern coupling.** Hardcoded string `:(exclude).dangeresque/runs` — if `RUNS_DIR` in `src/config.ts` ever changes, this drifts silently. Low risk (no reason to change it), but noting. A future refactor could import `CONFIG_DIR` and `RUNS_DIR` and compose the pathspec, at the cost of more moving pieces for one string.
