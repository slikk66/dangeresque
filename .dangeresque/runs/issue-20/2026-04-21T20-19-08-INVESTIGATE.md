## Skipped-review semantics (design decision)

Per issue's recommendation: set review_model/review_effort for non-skipped reviews; leave absent for skipped reviews.

In `ArtifactBuilder.build()`, the `review` local (lines 147-159) tells us exactly this:
- `review !== null && review.skipped === false` → review ran → INCLUDE both fields
- `review === null` OR `review.skipped === true` → review didn't run → OMIT both fields

Suggested build() patch (insert after `effort: this.init.effort ?? null,` at line 208):

```ts
const reviewRan = review !== null && !review.skipped;
// ...in the return object...
...(reviewRan && this.init.reviewModel !== undefined ? { review_model: this.init.reviewModel } : {}),
...(reviewRan && this.init.reviewEffort !== undefined ? { review_effort: this.init.reviewEffort } : {}),
```

This keeps new artifacts consistent with the migration rule below. Migration and new runs will both satisfy: "field present ⇔ review actually ran".

Concrete draft:

```js
#!/usr/bin/env node
import { readdirSync, readFileSync, writeFileSync, statSync } from "node:fs";
import { join } from "node:path";

function walk(dir) {
  const out = [];
  for (const name of readdirSync(dir)) {
    const p = join(dir, name);
    if (statSync(p).isDirectory()) out.push(...walk(p));
    else if (name.endsWith(".json")) out.push(p);
  }
  return out;
}

const root = process.cwd();
const files = walk(join(root, ".dangeresque", "runs"));
let migrated = 0, skipped = 0;

for (const file of files) {
  const a = JSON.parse(readFileSync(file, "utf-8"));
  if (a.schema_version !== "1") { skipped++; continue; }
  a.schema_version = "2";
  if (a.review && !a.review.skipped) {
    a.review_model = a.model;
    if (a.effort !== null && a.effort !== undefined) a.review_effort = a.effort;
  }
  writeFileSync(file, JSON.stringify(a, null, 2) + "\n", "utf-8");
  migrated++;
}
console.log(`Migrated ${migrated}, skipped ${skipped}.`);
```

## Risks / Uncertainty
