## Summary

Added `--issue-fixture <path>` to `dangeresque run`, following the plan from the prior INVESTIGATE run and the staged comment's Option A (`issueUrl?: string | null` on `BuilderInit`).

### Changes

1. **`src/runner.ts`** — new exported `loadIssueFixture(path: string): IssueData`.
   - Uses `node:fs.readFileSync` (already imported at the top of the file).
   - Validates: `number: number`, `title: string`, `body: string`, `comments: Array<{body:string, author:{login:string}, isMinimized:boolean}>`.
   - Throws clear `Error` with the path on: read failure, malformed JSON, wrong top-level type, missing required fields, bad comment shape.
   - Coerces `isMinimized` with `Boolean(...)` so fixtures can omit it (default false).

2. **`src/artifact.ts`** — added optional `issueUrl?: string | null` to `BuilderInit`. In `build()`:
   ```ts
   issue_url:
     this.init.issueUrl !== undefined
       ? this.init.issueUrl
       : buildIssueUrl(this.init.projectRoot, this.init.issueNumber),
   ```
   Explicit `null` is honored (the fixture case); undefined falls through to the existing remote-derived URL. Schema unchanged — `ARTIFACT_SCHEMA_VERSION` still `"1"`.

3. **`src/cli.ts`**:
   - Imported `loadIssueFixture` from `./runner.js`.
   - Added `--issue-fixture <path>` to `usageForEngine()` help text.
   - Parsed `--issue-fixture` into `issueFixturePath` in the arg loop.
   - Mutual-exclusion check after parse: `--issue` + `--issue-fixture` → `console.error` + `process.exit(1)`.
   - `fixtureUsed = issueFixturePath !== undefined` local, computed once.
   - When `issueFixturePath` set: `issueData = loadIssueFixture(...)`, `issueNumber = issueData.number`, prints `Loaded fixture #<n>: <title> (from <path>)`. On loader error → exit 1.
   - `ArtifactBuilder` construction: `...(fixtureUsed ? { issueUrl: null } : {})` spread — only introduces the key when fixture is used, preserves existing behavior otherwise.
   - Both `postRunComment` call sites (failure path at cli.ts:296 and success path at cli.ts:406) gated as `if (issueNumber && !fixtureUsed)`.

4. **`src/index.ts`** — added `loadIssueFixture` to the `./runner.js` re-export block alongside `fetchIssue` / `postRunComment`.

5. **`test/fixtures/example-issue.json`** — new file with the exact shape from the issue. 4 fields, empty comments array.

Commit: `57ab224 add --issue-fixture flag for offline runs without gh` (5 files, 107 insertions, 6 deletions).
