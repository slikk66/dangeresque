## Summary

`dangeresque logs` no longer auto-follows when the target worktree is RUNNING. `-f`/`--follow` is now the only way to tail. When the worker is RUNNING and `-f` was omitted, a one-line discoverability hint is printed to **stderr** after the snapshot completes, so stdout stays clean for pipes.

Implements the blueprint from the prior INVESTIGATE run (`2026-04-21T18-49-18-INVESTIGATE.md`). Exact locations in the issue hint ("line 422") were stale; actual site was `src/cli.ts:520`.

## Changes

### `src/cli.ts` — `cmdLogs`

1. `const follow = followFlag || target.running;` → `const follow = followFlag;`
2. After `await tailLog(...)`, append:

```ts
if (target.running && !followFlag) {
  console.error(
    "\nworker is RUNNING — pass -f/--follow to tail live output",
  );
}
```

The hint is skipped when `followFlag` is true (user is tailing — pointless) and when the target is IDLE (nothing to tail).
