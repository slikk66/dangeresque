## Fix Plan — Minimal Diff

**Scope**: `src/cli.ts:333-365` only. Two independent edits to the same block.

### A. Diff base: two-dot → three-dot (fixes Class 3)

Replace:
```ts
const changedFiles = execSync("git diff main --name-only", { ... });
```
With:
```ts
const changedFiles = execSync("git diff main...HEAD --name-only", { ... });
```

`git diff A...B` == `git diff $(git merge-base A B) B` — the diff from merge-base to HEAD. Excludes commits added to main after the worker's branch-point. Drop-in replacement, no new `execSync` call needed.

Edge case check: scope-check runs before the rebase at L369-390, so at this point HEAD is still on the original branch-point and the merge-base is stable. Safe.

### B. Haystack: body → body + staged comments (fixes Class 1)

Option B1 (recommended, ONE-PATH): export `formatIssueComments` from `src/runner.ts` and reuse it. Haystack:

```ts
import { ..., formatIssueComments } from "./runner.js";

const haystack = issueData.body + formatIssueComments(issueData);
const unexpected = changedFiles.filter((f) => !haystack.includes(f));
```

Mirrors the exact text the worker reads as task input. Zero duplication of the filter predicate — changes to `formatIssueComments` (e.g., changing `**[staged` marker) automatically flow to the scope-check.

Option B2 (explicit staged-only): duplicate the filter inline. Less ONE-PATH-friendly; rejected.

Export change needed in `src/runner.ts:153`:
```diff
-function formatIssueComments(issueData: IssueData): string {
+export function formatIssueComments(issueData: IssueData): string {
```

**Chosen: B1.** Satisfies the ONE-PATH constraint from the issue ("reuse the same staged-comment filter that `formatIssueComments` uses (don't invent a new matcher)").
