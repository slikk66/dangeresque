## Changes

### `src/runner.ts` (src/runner.ts:299-314 new helper; src/runner.ts:339, 397, 456, 499 call sites)

1. **New export** (immediately above `buildClaudeWorkerArgs`):
   ```typescript
   export function readPromptWithLocal(configDir: string, baseName: string): string {
     const canonical = readFileSync(join(configDir, baseName), "utf-8");
     const localName = baseName.replace(/\.md$/, ".local.md");
     const localPath = join(configDir, localName);
     if (existsSync(localPath)) {
       const local = readFileSync(localPath, "utf-8").trim();
       if (local.length > 0) return canonical + "\n\n" + local;
     }
     return canonical;
   }
   ```

2. **`buildClaudeWorkerArgs`** — removed the `workerPromptPath` local; flag swap:
   ```typescript
   -  const workerPromptPath = join(configDir, config.workerPrompt);
   -  args.push("--append-system-prompt-file", workerPromptPath);
   +  args.push("--append-system-prompt", readPromptWithLocal(configDir, config.workerPrompt));
   ```

3. **`buildClaudeReviewArgs`** — symmetric:
   ```typescript
   -  const reviewPromptPath = join(configDir, config.reviewPrompt);
   -  args.push("--append-system-prompt-file", reviewPromptPath);
   +  args.push("--append-system-prompt", readPromptWithLocal(configDir, config.reviewPrompt));
   ```

4. **`buildCodexWorkerArgs`** — replaced the raw `readFileSync` with the helper; introduced `configDir` local for clarity:
   ```typescript
   -  const workerPromptPath = join(opts.projectRoot, CONFIG_DIR, opts.config.workerPrompt);
   -  const workerPromptContent = readFileSync(workerPromptPath, "utf-8");
   +  const configDir = join(opts.projectRoot, CONFIG_DIR);
   +  const workerPromptContent = readPromptWithLocal(configDir, opts.config.workerPrompt);
   ```

5. **`buildCodexReviewArgs`** — symmetric.

### `test/unit/runner.test.ts`

Added `readPromptWithLocal` to the existing `#dist/runner.js` import block, then appended 5 tests using the established `mkdtempSync` + real-fs pattern:

- **canonical only, .local.md missing → canonical** (most common current case; zero `.local.md` files exist today).
- **canonical + .local.md with content → canonical + "\n\n" + trimmed local** (the actual feature; explicit concatenation assertion: `"CANON\n\n\nLOCAL ADDITION"` — three newlines because canonical's trailing `\n` plus the separator's `\n\n`).
- **canonical + empty .local.md → canonical only** (trim check on `""`).
- **canonical + whitespace-only .local.md → canonical only** (trim check on `"   \n\n  \t\n"`).
- **canonical missing → throws ENOENT** (error propagation unchanged).
