# Issue #31 — IMPLEMENT

## Status

implemented, unverified

## Summary

Surfaces review coverage as an explicit derived metric and tightens skipped-verdict
semantics across three user-facing surfaces: Summary output, CLI help, and the
glossary + README evaluation vocabulary.

### Changes

1. **`src/stats.ts`**
   - Added `reviewCoveragePercent: number` to `StatsSummary` interface (additive;
     sits alongside `byModeDurations`).
   - `computeStats` populates it: `total > 0 ? (1 - byVerdict.skipped / total) * 100 : 0`.
   - One-line comment notes that `unknown` verdicts are lumped with "reviewed"
     so `reviewed + skipped = total` holds (arithmetically self-consistent output).
   - `formatSummary` replaces the old `Review coverage: X% (R/T) runs reviewed` line
     with `Review coverage: X% (R/T runs reviewed, S skipped)`. Neutral wording
     "skipped" (not "auto-skipped") because `--no-review` runs land in the same bucket.
     Uses `reviewCoveragePercent` as the source for the percentage (one number, one source).

2. **`src/cli.ts`**
   - `usageForEngine`: extended the `stats` line to mention the auto-skip rule.
     New text: `Aggregate run evaluation artifacts; review auto-skipped for INVESTIGATE/VERIFY modes; --glossary explains terms`.
   - `statsGlossary()`: tightened the `skipped` verdict entry. Was:
     "No reviewer decision exists because review was intentionally skipped."
     Now: "The reviewer did not run. Automatically skipped for INVESTIGATE and VERIFY modes (no code changes to audit), and manually skipped by --no-review."

3. **`README.md`**
   - Tightened `reviewer_verdict=skipped` line in the Evaluation Vocabulary
     section to match the glossary phrasing: "The reviewer did not run
     (INVESTIGATE/VERIFY modes auto-skip; `--no-review` forces skip)."

4. **`test/unit/stats.test.ts`**
   - Updated exact-string Summary-line assertion to the new format:
     `Review coverage:     75.0% (3/4 runs reviewed, 1 skipped)`.
   - Updated empty-data regex:
     `/Review coverage:\s+0\.0% \(0\/0 runs reviewed, 0 skipped\)/`.
   - Added new test `computeStats: reviewCoveragePercent = (1 - skipped/total) * 100`
     covering four cases: total=0, all accepted (100), all skipped (0), mixed 12+13/25
     (48, asserted via `.toFixed(1)` to avoid FP noise), and unknown-lumped-with-reviewed.
