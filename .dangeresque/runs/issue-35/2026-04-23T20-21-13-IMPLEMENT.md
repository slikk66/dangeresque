## Risks / Uncertainty

- **`child.stdin.end(prompt)` pattern.** Using `.end()` with a single-argument write is the canonical Node idiom ‚Äî equivalent to `write(prompt); end();` ‚Äî and flushes atomically before closing. No risk of codex blocking on `read_to_end`.
- **TTY detection.** Node's `"pipe"` fd is a non-TTY pipe on Unix; codex's `is_terminal()` returns false, so `-` path executes `read_to_end` without a terminal-error bailout. Safe on macOS/Linux (the only platforms dangeresque supports).
- **Backpressure.** Prompts bounded to ~10‚Äì50 KB by issue body + comments + prompt templates. Well under the default 16 KB+ pipe buffer + Node's writable highWaterMark. `child.stdin.end()` internally handles drain; no explicit backpressure handling needed.
- **Scope containment.** Only `src/runner.ts` (codex branches) and `test/unit/runner.test.ts` touched. No changes to config schema, CLI surface, artifact format, claude paths, or prompt templates.
